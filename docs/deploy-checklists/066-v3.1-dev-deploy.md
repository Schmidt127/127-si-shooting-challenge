## Test plan (one enrollment)

Use a **test enrollment** (Schmidt or one of the five retained test enrollments) that has:

- **Grade Band** linked
- **Active?** — script skips inactive; use **checked** for milestone test (or confirm skip path separately)
- Linked **Submissions** with **Count This Submission?** and **Total Shots Counted** summing past at least one **Shot Milestone** threshold for that grade band

### Steps

1. Note enrollment `rec…` and current unlock count for that enrollment.
2. Check **Run Shot Milestone Check?** on the enrollment (or use existing trigger path).
3. Wait for automation run → open run history.
4. **Pass if:**
   - `statusOut` = `success` (or `skipped` with expected `actionOut` e.g. `skipped_existing`)
   - `errorOut` empty
   - `debugStep` reaches final step
   - Console JSON shows `"version": "v3.1"`
   - New **Athlete Achievement Unlock** rows have **Milestone Source Key** `SHOT_MILESTONE|{enrollmentId}|{shotMilestoneId}`
   - **Week** populated on new unlocks when **Milestone Activity Date** maps to a **Weeks** row
   - **Run Shot Milestone Check?** cleared on success/skip (not left checked on error)
5. Re-check same enrollment (idempotent): second run should **not** duplicate unlocks (`skipped_existing` or zero creates).

### Idempotency re-run

```
[ ] Second trigger on same enrollment → no duplicate unlock rows (same Milestone Source Key)
```
